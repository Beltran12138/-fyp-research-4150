## 5.8.1 llms.txt — AI Crawler Context File

**Deployed:** [beltran12138.github.io/hkx/llms.txt](https://beltran12138.github.io/hkx/llms.txt)  
*(Production deployment: hashkey.com/llms.txt)*

The `llms.txt` standard (llmstxt.org, 2025) provides AI language models with a structured, human-readable summary of a site's identity, authority signals, and canonical content URLs — enabling citation-ready context without requiring full site crawl. It is the AI-era equivalent of `robots.txt`: a single root-level file that declares what the site is and what its most important content is.

**Design rationale for this campaign:** Three design decisions distinguish this `llms.txt` from a generic brand summary. First, the opening blockquote encodes the two facts most likely to be cited in a "which exchange is safe?" query: SFC licensing and HKEX listing. Second, the Bitcoin Pizza Day section explicitly names the campaign URL (`/hkx/rib/`) as a canonical source — anchoring the RIB microsite as the institutional record of the event for AI crawlers. Third, developer resources (SkillHub, hkx CLI, KYA Agent Registry) are listed under a dedicated `## Developer Resources` heading, giving the developer-segment queries a direct citation pathway separate from the consumer compliance narrative.

## 5.8.2 JSON-LD Schema Markup

**Deployed pages:**
- RIB microsite: [beltran12138.github.io/hkx/rib/](https://beltran12138.github.io/hkx/rib/) — `Event` + `FAQPage` schemas
- SkillHub: [beltran12138.github.io/hkx/](https://beltran12138.github.io/hkx/) — `FinancialService` (Organization) schema

Three schema types are injected into `<head>` on campaign-relevant pages. JSON-LD is chosen over Microdata because it keeps structured data entirely separate from HTML — enabling deployment on static GitHub Pages without modifying content structure.

**Organization schema (SkillHub):** Uses `FinancialService` rather than the generic `Organization` type. This specificity matters: `FinancialService` is the schema type that triggers the "financial institution" entity recognition in Gemini's knowledge graph, which governs how the exchange is described when users query "which exchanges are regulated in HK?" The `hasCredential` field explicitly names the SFC VATP licence category — the primary differentiator from unregulated offshore competitors.

**Event schema (RIB microsite):** Marks 22 May 2026 as a HashKey-hosted event with a defined end time of 20:10 HKT — the airdrop trigger moment. This creates a durable machine-readable record that positions HashKey as the institutional actor in Hong Kong's Bitcoin Pizza Day 2026 narrative. Post-campaign, the schema continues to function as a historical record, reinforcing HashKey's association with the date in future AI queries.

**FAQPage schema (RIB microsite):** Four question-answer pairs calibrated against the three query-intent classes from Section 4.5.6 (Buy / Solve / Learn) plus the campaign-specific "What is RIB?" question. FAQPage schema achieves 3.2× higher AI citation rates versus unstructured equivalent pages (Insightland, 2025). The four questions are also aligned with the Laszlo Bot's intent classification taxonomy (Section 5.1), ensuring consistent brand voice across human-facing and AI-facing content surfaces. Answers are kept within 80 words — the FAQPage specification limit for rich result eligibility.

## 5.8.3 Quick Answer Content Blocks

**Deployed:** [beltran12138.github.io/hkx/rib/](https://beltran12138.github.io/hkx/rib/) (FAQ section, pre-footer)

Three structured content blocks are published as visible HTML on the RIB microsite, formatted for direct extraction by AI answer engines. The format follows the ≤40-word direct answer + source attribution pattern that Perplexity and ChatGPT preferentially surface (Frase, 2025).

Each block addresses one of the three query-intent classes identified in Section 4.5.6:

- **Buy intent** ("Which exchange should I use in HK?") — positions HashKey as the only SFC-licensed retail option, cites specific licence types, links to hashkey.com for KYC
- **Solve intent** ("Is HashKey safe?") — marshals the four trust signals (SFC regulation, ISO certification, HKEX listing, operational continuity through three market crises) into a single ≤40-word block
- **Learn intent** ("What is Bitcoin Pizza Day?") — anchors HashKey to the 22 May 2026 date, cites the 10,001st Slice airdrop as the 2026 campaign record

The blocks are rendered as visible page content (not hidden via CSS) to satisfy AI extraction requirements: content hidden with `display:none` is deprioritised by crawlers. The dark-theme styling is applied via `.qa-item` CSS classes that match the RIB microsite's visual register without disrupting the memorial wall experience.

The distinction between FAQPage JSON-LD (5.8.2) and Quick Answer blocks (5.8.3) is intentional: JSON-LD is machine-readable structured data targeting crawler-phase ingestion; Quick Answer blocks are natural-language content targeting inference-phase extraction. Both signal layers must be present because different AI engines rely on different extraction mechanisms at different phases of the citation pipeline.
