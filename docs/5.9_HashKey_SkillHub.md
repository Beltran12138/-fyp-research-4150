## Concept

AI agents now account for 58% of global crypto trading volume (FinanceMagnates, 2026). As agentic adoption accelerates, every exchange faces the same question: what happens when an AI agent, not a human, tries to trade on your platform?

Most exchanges answer with silence. HashKey answers with infrastructure.

**HashKey SkillHub** is the first SFC-framework-compatible agent skills registry for Hong Kong retail crypto. It is a curated marketplace of AI trading skills — built on the agentskills.io open standard (compatible with Claude Code, Codex, Cursor, and Hermes Agent) — where every listed skill has passed HashKey's compliance review against SFC guidelines before deployment.

**The Safe Punk argument in technical form:** anyone can build an AI trading bot. Only HashKey builds one that had to pass the same scrutiny as a licensed exchange.

---

## How It Works

Skills follow the [agentskills.io](https://agentskills.io) open standard — Markdown procedure files with a compliance metadata header added by HashKey.

**Skill lifecycle:**
```
Developer submits skill → HashKey compliance review (SFC guidelines) 
→ Approved / Rejected → Listed on SkillHub → Agent installs via CLI or API
```

**Install a skill (CLI):**
```bash
hkx skill install hashkey-dca
```

**Install via Claude Code / Codex:**
```
/install hashkey:hashkey-dca
```

## Sample Skill File

```markdown
---
name: hashkey-dca
version: 1.2.0
author: HashKey Exchange
compliance: SFC-reviewed
approved_assets: [BTC, ETH, USDT, USDC, HKEX-listed tokens]
prohibited: [price-prediction, investment-advice, unlicensed-asset-trading]
last_reviewed: 2026-04-01
---

# HashKey DCA Skill

Executes a dollar-cost averaging strategy on HashKey Exchange 
within SFC-approved asset parameters.

## Trigger
User sets: asset, amount per interval, interval frequency.

## Steps
1. Verify target asset is in `approved_assets` list
2. Check user account KYC status via HashKey API
3. Place limit order at current mid-price ± 0.1%
4. Log trade with timestamp and confirmation ID
5. Report execution summary to user

## Constraints
- Never recommend which asset to DCA into
- Never execute trades outside `approved_assets`
- Never exceed user-defined per-interval limit
- If KYC inactive → pause and notify user

## Example output
> DCA executed: 0.0012 BTC purchased at HKD 742,310.
> Next execution: 2026-05-22 09:00 HKT.
> All trades logged to hashkey.com/account/history.
```
